## 分析

### 迭代性能对比

```
fib(100000) 迭代性能:
┌─────────────┬──────────┐
│ Go          │   0 ms   │ ████████████ 最优

│ Python      │  60 ms   │ █
└─────────────┴──────────┘
```

**关键发现:**

1. **小规模(n≤10000)**: 三者都很快，差异不明显
2. **大规模(n=100000)**:
   - Go: 0ms - 编译型语言优势明显
   - Charlang: 6ms - 性能接近Go，表现优秀
   - Python: 60ms - 比Charlang慢10倍

3. **Charlang vs Python (迭代)**:
   - 迭代fib(100000): Charlang比Python快 **10倍**

## 结论

### Charlang 性能亮点
- ✅ 迭代性能接近Go
- ✅ 迭代比Python快10倍
- ✅ 大数运算效率高
- ⚠️ 递归调用有额外开销

### 建议
- **优先使用迭代而非递归** - 在所有语言中都成立
- Charlang适合数值计算场景
- 大规模数据处理(迭代)表现优秀
- 递归场景应控制深度或改用迭代

### 递归支持说明
Charlang支持递归函数，需使用特殊语法:
```charlang
var fib
fib = func(n) {
    if n <= 1 { return n }
    return fib(n-1) + fib(n-2)
}
```
而非 `fib := func(n) {...}` 短声明形式。
